# HN Post Workshop — Vessel Room Navigator

## Audience Split

**Engineers** will ask: "What problem does this solve? How is it different? Show me the code."
**Frontend devs** will ask: "How did you build that? Can I fork it? What's the stack?"

Both need a-ha moments. Neither wants to be told what to think.

## Angle 2: "Architecture First" (hook with insight, then demo)

**Title:** Rooms are the new prompts

**Opening:**
"Every AI system today follows the same pattern — one model, one prompt, one response. If you want it to do something complex, you make the model bigger. But bigger models hallucinate more, cost more, and still can't do multi-step reasoning reliably.

We tried something different. Instead of one giant model that knows everything, we built rooms. Each room is a constraint boundary — a space where only certain things are relevant. The engine room has temperature gauges and thermal cameras. The wheelhouse has radar and navigation charts. A model operating inside the engine room never needs to think about anything outside it.

We built a prototype to prove it: a fishing vessel as a navigable 3D web space. https://fleet.cocapn.ai/"

**a-ha moment 1** (engineer): "Small model + good structure beats large model + no structure"
**a-ha moment 2** (frontend): "Three.js panoramas from AI images, 38KB, no build step"

## Angle 3: "The $0.03 Post" (hook with cost, then everything else)

**Title:** I built a 3D walkable fishing boat for $0.03 (and learned something about AI architecture)

**Opening:**
"The panoramas in this demo cost three cents to make. All seven of them, 1792×1024 pixels, generated by FLUX-1-schnell. The viewer is a single HTML file with Three.js from a CDN. Open it, drag around, warp between rooms. No install, no build step, no backend.

https://fleet.cocapn.ai/

But the real story isn't the three cents. It's what the boat revealed about AI architecture.

We've been building a system where small models in well-structured rooms outperform large models with no structure. Forgemaster's FLUX runtime proved it on real hardware — a Python implementation running on one CPU core (84ns per operation) beat a C implementation with full compiler optimization (256ns) for small primitives. The overhead of crossing a language boundary cost more than the computation itself.

The room structure WAS the intelligence. The model just followed it.

The boat is that principle made visible."

**a-ha moment 1** (everyone): "Three cents for seven AI panoramas"
**a-ha moment 2** (engineer): "Structure replaces model size"
**a-ha moment 3** (both): "The demo is THE architecture, not a separate thing"
