# docs/release/hn-post.md

A single-file 3D room viewer for a fishing vessel. Seven AI-generated 360° panoramas connected by doors and warps. Drag to look around. Open in any browser — it's 38KB of HTML with Three.js from a CDN.

https://fleet.cocapn.ai/

---

The panoramas were generated by FLUX-1-schnell. Each is 1792×1024. All seven cost $0.03 in API credits.

The viewer has no build step. No install. No backend. Open the page and walk around.

---

The rooms are more than visual. They're constraint boundaries.

Each room defines what's relevant: the engine room has temperature gauges and thermal cameras; the wheelhouse has radar and nav charts. A model operating inside a room doesn't need to know what's outside it — the room defines the context.

We built a runtime that proves this works. It probes the system, discovers compilers, compiles the same kernel in C, Python, Fortran, Zig, and Nim, benchmarks each, and picks the fastest. On real hardware, a Python implementation (84ns per op) beat C with full optimization (256ns) for small primitives. The overhead of crossing a language boundary cost more than the computation. The structure of the room — small primitive, known interface, local execution — was the intelligence. The model just followed it.

The 3D viewer is the same principle made visible. The rooms are the structure. The model navigates them.

---

Three of us built room systems independently this week — a 3D boat, a MUD-to-3D bridge, and a constraint theory workspace. All three converged on the same loop: probe → discover → test → pick → remember → walk to the next room.

That's the architecture. The boat is a demo of it.
