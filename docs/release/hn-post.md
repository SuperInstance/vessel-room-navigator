You're standing in the wheelhouse of a fishing boat. Warm light from the radar screens. Teak trim. A coffee mug in a gimbal mount, half full. You can see the whole room by dragging your mouse — 360°, no seams, no loading.

Press 7 and you're in the crow's nest, looking out at the Bering Sea from 40 feet up.

This is a single HTML file. 38 kilobytes. Three.js from a CDN. The panoramas were generated by FLUX-1-schnell — all seven of them, 1792×1024, cost about three cents total in API credits.

Open it in any browser. No install. No build step. No backend. The page is at https://fleet.cocapn.ai/ and the entire source is at github.com/SuperInstance/vessel-room-navigator.

That's the demo. But the demo is not the point.

---

**Why would anyone build this?**

Because every AI system today follows the same pattern: one giant model, one prompt, one response. If you want it to do more, you make the model bigger. Bigger models hallucinate more. They cost more. They still can't do reliable multi-step reasoning. The whole industry is scaling parameters because nobody designed a different pattern.

**What's the alternative?**

Build rooms. Each room is a constraint boundary — it defines what exists, what normal looks like, what actions are valid. The engine room has temperature gauges and thermal cameras. The wheelhouse has radar and charts. A model operating inside the engine room never needs to think about anything outside it. The room defines the context. The model just navigates.

The boat is a literal example. Digital rooms work the same way.

**Where's the proof?**

A runtime we built probes the system, discovers compilers, compiles the same kernel in C, Python, Fortran, Zig, and Nim, benchmarks each, and picks the fastest. What it found: a Python implementation (84 nanoseconds per operation) beat C with full compiler optimization (256 nanoseconds) for small primitives.

The overhead of crossing a language boundary — the function call, the marshaling, the context switch — cost more than the computation itself. The room structure was the intelligence. The model just followed it.

**Three teams built room systems independently this week.** Oracle1 built the 3D boat. CCC built a MUD-to-3D bridge. Forgemaster built constraint theory workspaces. All three converged on the same loop: probe → discover → test → pick → remember → walk to the next room. Nobody coordinated this. The pattern emerged because it's the right shape.

**Who uses this?**

We do. The fleet runs 24/7 — 12 agents on a loop, filing knowledge tiles to a shared graph called PLATO, routing work between rooms through a small model that decides and a large model that executes. The small model costs near nothing. The large model only runs when precision matters.

The demo is the same architecture, visible. The boat is what the inside looks like from the outside.

**How do I try it without feeling like it's a toy?**

Open the link. Drag around the wheelhouse. Press 2 for the galley. Press 7 for the crow's nest. Click the side panel and trigger an alarm — you'll warp to the problem room.

Then clone the repo. The research in docs/research/ covers room topology analysis (Laman rigidity), camera architecture (5 types × 5 modes), GPU vector databases (5 backends, auto-detecting), and the mathematical foundations (zero-holonomy consensus, Pythagorean48 encoding).

The README tells you how to serve it locally. It's a Python one-liner.

**How do I customize this for my own use?**

The room definitions are inline JavaScript objects. Each room has: name, type, adjacencies, cameras, dashboards. You can add rooms, change textures, wire in your own sensors. The research docs describe how to connect it to PLATO (the knowledge graph), ESP32 camera agents, and the keel CLI for fleet management.

The architecture is designed to be forked, not to be a platform.

A hermit crab outgrows its shell. It doesn't break the old one — it finds a new one. The old shell becomes a home for the next crab. Nothing is wasted. The beach accumulates better shells over time.

Every agent leaves its shell better than it found it. The next agent inherits everything the previous one learned. The shell outlives every inhabitant.

We built the shell. The boat is the shell. The repos are the shells. The architecture is the shell — constraint boundaries that outlive the models that navigate them.
